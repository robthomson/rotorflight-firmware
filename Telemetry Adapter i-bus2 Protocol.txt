Telemetry Adapter i-bus2 Protocol
Definition
1. Explanation
This protocol is based on the i-bus2 basic protocol:
For the i-bus2 general protocol, please refer to the i-bus2 protocol document.
The following is the protocol breakdown definition of the telemetry adapter.
The telemetry adapter is the slave device, and will be referred to as the slave device below.
The receiver is referred to as the master device.
2. Master command
data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned char Data[19];
unsigned char CRC8;
};
CommandCode is the command code, and will be explained according to the command code
below
2.1 CommandCode is 3
The master device obtains the device parameters.
Data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned short ParamType
unsigned char Reserved[17];
unsigned char CRC8;
} sIB2_CommandGetParam;
For example: Inquire whether the slave device supports advanced functions. The function
code for advanced functions is 0x01
The data protocol is: (see the appendix for the specific protocol encoding)
0x01(2b) 0x03(6b) 0x01 0X00 0X00 0X00 0X00 …… CRC
2.2 CommandCode is 4
The host sets the device parameters.
Data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned short ParamType;
unsigned char ParamLength;
unsigned char ParamValue[16];
unsigned char CRC8;
} sIB2_CommandSetParam;
For example: Set the slave's working protocol number to protocol number 5 (see appendix
for specific protocol codes)
The data protocol is:
0x01(2b) 0x04(6b) 0x03 0X00 0X01 0X05 0X00 …… CRC
2.3 CommandCode is for other cases
Other cases are the standard i-bus2 protocol part, which will not be elaborated on.
3. Slave command
data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned short Data[19];
unsigned char CRC8;
};
CommandCode is the command code. The following explanation is based on the command
code
3.1 When CommandCode is 3
the device replies to the host with the command to obtain device parameters, and the device
returns the specific value of the parameter.
Data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned short ParamType;
unsigned char ParamLength;
unsigned char ParamValue[16];
unsigned char CRC8;
} sIB2_ResponseGetParam;
For example: the responding host supports advanced functions. The function code of the
advanced function is 0x01
The data protocol is: (see the appendix for the specific protocol encoding)
0x02(2b) 0x03(6b) 0x01 0X00 0X01 0X01 0X00 …… CRC
3.2 CommandCode is 4
The device responds to the host with commands to set device parameters, returning a success
or failure message.
Data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned short ParamType;
unsigned char ParamLength; // 0 if parameter not supported
unsigned char Reserved[16];
unsigned char CRC8;
} sIB2_ResponseSetParam;
For example: the result of the response host setting protocol number 5 (see appendix for
specific protocol codes).
The data protocol is as follows:
0x02(2b) 0x04(6b) 0x03 0X00 0X01 0X01 0X00 …… CRC
3.3 CommandCode is 2
The device replies to the host with a command to retrieve data from the device (the data
returned by the device will be received as sensor data and returned to the transmitter (such
as voltage, current, speed, etc.)). The maximum length is 14.
Data format
typedef __packed struct
{
unsigned char PacketType:2;
unsigned char CommandCode:6;
unsigned char Value[14];
unsigned char VID;
unsigned char PID;
unsigned char Reserved[3];
unsigned char CRC8;
} sIB2_ResponseGetValue;
Definition of Value[14]:
unsigned char PackType:4 data type
unsigned char PackLenght:6 Total number of data packets
unsigned char PackCurIndex:6 The sequence number of the currently sent packet.
unsigned char Data1Type:6 See details: Return Parameter Name (Sensor Type)
For details on unsigned char Data1Format:5, please see: Data Format
For details on unsigned char Data1Unit:5, please see: Data Units
The unsigned short Data1 data portion transmits S16 type data normally.
unsigned char Data2Type:6 See details: Return Parameter Name (Sensor Type)
For details on unsigned char Data2Format:5, please see: Data Format.
For details on unsigned char Data2Unit:5, please see: Data Units.
The unsigned short Data2 data portion transmits S16 type data normally.
unsigned char Data3Type:6 See: Return Parameter Name (Sensor Type) for details.
For details on unsigned char Data3Format:5, please see: Data Format.
For details on unsigned char Data3Unit:5, please see: Data Units.
The unsigned short Data3 data portion transmits S16 type data normally.
For example, the current response from the host only contains a voltage data point: 5.25V
(see appendix for specific protocol encoding).
The data protocol is as follows:
0x02(2b) 0x02(6b) 0x00(4b) 0X01(6b) 0X01(6b) 0X01(6b) 0X02(5b) 0X01(5b)
0X0B 0X02 0X00(6b) 0X00(5b) 0X00(5b) 0X00 0X00 0X00(6b)
0X00(5b) 0X00(5b) 0X00 0X00 VID PID … CRC
For example, the current response to the host contains four data points: a voltage of 5.25V, a
current of 0.5A, a speed of 60,000 m/s, and a signal strength of 100 (see the appendix for
specific protocol encoding).
The data protocol is as follows:
First package
0x02(2b) 0x02(6b) 0x00(4b) 0X02(6b) 0X01(6b) 0X01(6b) 0X02(5b) 0X01(5b)
0X0B 0X02 0X02(6b) 0X01(5b) 0X02(5b) 0X05 0X00 0X00(6b)
0X00(5b) 0X00(5b) 0X60 0XEA VID PID … CRC
Second package
0x02(2b) 0x02(6b) 0x00(4b) 0X02(6b) 0X02(6b) 0X18 (6b) 0X00(5b) 0X00(5b)
0X64 0X00 0X00(6b) 0X00(5b) 0X00(5b) 0X00 0X00 0X00(6b)
0X00(5b) 0X00(5b) 0X00 0X00 VID PID … CRC
This data is normal, real-time sensor-related data. If the data exceeds the number of packets
to be sent, it will be sent in multiple packets. If a host-initiated request command is inserted
midway, the host command will be responded to first, and then the loop will continue.
3.4 CommandCode for other cases
In other cases, it follows the standard i-bus2 protocol, which will not be elaborated further.
4. Product ID
VID:0X01
PID:0X03
5. Command Description
Special instructions are provided for certain commands.
5.1. Advanced commands
Advanced commands are one of the key features of this release.
We need to implement an advanced command that queries whether a command is supported
and returns all supported commands.
After using the advanced command to query the host, the returned data format is referenced
in section 3.1.
typedef __packed struct
{
unsigned char PacketType:2; //0X02
unsigned char CommandCode:6; //0X03
unsigned short ParamType; //0X0001
// Supports a maximum of 16 advanced commands. Returning 0 or a timeout indicating no
response means it is not supported.
unsigned char ParamLength;
//The order represents the range of values to be sent for the corresponding supported
advanced commands.
If two advanced commands are supported, with a length setting of 2, the first command
supports sending parameters 0 and 1, while the second command supports sending
parameters 0 to 4.
Then ParamLength = 2, ParamValue[0] = 2, ParamValue[1] = 5, and the rest are padded
with 0.
unsigned char ParamValue[16];
unsigned char CRC8;
} sIB2_ResponseGetParam;
